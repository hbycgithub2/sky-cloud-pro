# 如何使用模板生成新技术文档

> 会话断了也能继续，按模板填空即可

---

## 🎯 使用场景

### 场景1：会话断了，需要继续
1. 打开 `模板-精简版.md` 或 `模板-详细版.md`
2. 复制模板内容
3. 替换 `[技术名称]` 等占位符
4. 保存为新文件

### 场景2：自己整理新技术
1. 选择模板（精简版或详细版）
2. 按模板结构填写内容
3. 保存为对应文件名

### 场景3：让AI帮你生成
1. 告诉AI："用模板生成Nginx文档"
2. AI按模板结构生成内容
3. 你确认后保存

---

## 📋 模板使用步骤

### 步骤1：选择模板

**精简版（模板-精简版.md）**
- 适合：面试、快速查阅、日常工作
- 结构：10秒版 + 技术要点 + 深度解析

**详细版（模板-详细版.md）**
- 适合：深入学习、实际开发、问题排查
- 结构：5层递进（主旨 + 核心价值 + 实际案例 + 问题解决 + 关键配置）

---

### 步骤2：替换占位符

**需要替换的占位符：**

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `[技术名称]` | 技术的名称 | Gateway、Nginx、Token |
| `[问题1]` | 解决的问题 | 前端多地址问题 |
| `[端口号]` | 技术使用的端口 | 10010、8005 |
| `[配置内容]` | 关键配置代码 | routes配置 |
| `[步骤1]` | 流程步骤 | 前端请求到达 |
| `[日期]` | 更新日期 | 2026-01-29 |

---

### 步骤3：填写内容

**按模板结构填写：**

#### 精简版填写顺序：
1. **10秒版** - 核心亮点（3句话）
2. **技术要点** - 定义、功能、配置
3. **深度解析** - 实际案例、问题解决

#### 详细版填写顺序：
1. **主旨** - 核心中的核心（1-2句话）
2. **核心价值** - 解决什么问题
3. **实际案例** - 你的项目怎么用的
4. **问题解决** - 遇到什么坑，怎么解决的
5. **关键配置** - 拿来就能用

---

### 步骤4：保存文件

**文件命名规则：**
```
精简版：[技术名称].md
详细版：[技术名称]-详细版.md

示例：
Nginx.md
Nginx-详细版.md

Token.md
Token-详细版.md
```

## 🚀 快速示例

### 示例：生成Nginx文档

#### 1. 打开模板
```
打开：模板-精简版.md
```

#### 2. 替换占位符
```
[技术名称] → Nginx
[端口号] → 8005
[问题1] → 隐藏内部端口
[问题2] → 静态资源缓存
[问题3] → 安全防护
```

#### 3. 填写内容
```
10秒版：
Nginx反向代理，解决三大问题：
1. 隐藏内部端口（只暴露8005）
2. 静态资源缓存（快10倍）
3. 安全防护（限流/黑名单）

技术要点：
定义：Nginx反向代理服务器，边缘网关层，端口8005
核心功能：
- 反向代理：转发请求到Gateway
- 静态缓存：图片/CSS/JS直接返回
- 负载均衡：多台Gateway分流
- 安全防护：限流、黑名单、日志

关键配置：
upstream gateway_backend {
    server localhost:10010;
}
location /api/ {
    proxy_pass http://gateway_backend/;
}
```

#### 4. 保存文件
```
保存为：Nginx.md
```
